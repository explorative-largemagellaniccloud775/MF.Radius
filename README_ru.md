# MF.Radius 🛰️
[🇺🇸 English](https://github.com/morfair/MF.Radius/blob/main/README.md) | **Русский язык**

**MF.Radius** - это высокопроизводительный фреймворк для реализации протокола RADIUS на платформе **.NET 10**.
Спроектирован для работы в условиях экстремальных нагрузок (AAA-трафик) 
с акцентом на **Zero-Allocation** архитектуру и принципы **Clean Architecture**.

В репозитории представлены два проекта:
- **MF.Radius.Core** - ядро фреймворка, содержащее высокоуровневые абстракции и низкоуровневую логику работы с протоколом;
- **MF.Radius.SampleServer** - пример реализации RADIUS-сервера, демонстрирующий лучшие практики использования библиотеки.

## 🚀 Что делает проект?
**MF.Radius** предоставляет инструментарий для создания собственных RADIUS-серверов.
Библиотека берет на себя всю сложную работу по парсингу, валидации и сборке пакетов:
- **Аутентификация**: полная поддержка циклов `Access-Request`, `Accept`, `Reject`;
- **Аккаунтинг**: обработка запросов `Accounting-Request` и подтверждений `Response`;
- **Управление сессиями**: поддержка динамических запросов `CoA-Request` и `Disconnect-Request` (DM);
- **Криптография**: встроенная реализация алгоритмов `PAP`, `CHAP` и `MS-CHAPv2`.

## ✨ Почему проект полезен?
В современных сетевых решениях каждая микросекунда и каждая пауза сборщика мусора (GC) 
напрямую влияют на пропускную способность системы.
- **Производительность**: критические пути обработки данных (`hot paths`) оптимизированы 
с использованием `Span<T>`, `Memory<T>`, `ArrayPool` и `MemoryPool`, что сводит аллокации в куче к минимуму и снижает нагрузку на CPU;
- **Гибкость и расширяемость**: архитектура позволяет легко добавлять поддержку специфичных атрибутов (VSA) 
для оборудования Cisco, MikroTik и любых других вендоров;
- **Чистая архитектура**: в Sample-сервере показано, как библиотека бесшовно встраивается в проекты, построенные на принципах **SOLID**, **DDD** и **CQRS**;
- **Современная экосистема**: вы не ограничены в выборе инфраструктуры,
используйте любые БД, кэши, брокеры сообщений и инструменты мониторинга (OpenTelemetry);
- **Готовый шаблон**: Sample-сервер можно использовать как готовый скелет для вашего Enterprise-решения, 
не тратя время на проектирование базовой структуры.

## 🏁 Как приступить к работе?

### 1. Установка зависимости
Пакет будет доступен в [NuGet](https://www.nuget.org/packages/MF.Radius.Core) после первого публичного релиза.
Для установки нужно будет выполнить команду:
```bash
dotnet add package MF.Radius.Core
```

### 2. Выбор способа интеграции
Вы можете интегрировать библиотеку в свой проект на разных уровнях абстракции:
- **Использование шаблона**: возьмите за основу [MF.Radius.SampleServer](https://github.com/morfair/MF.Radius/tree/main/src/MF.Radius.SampleServer).
Вся бизнес-логика в нем сосредоточена в классе `IspRadiusProcessor`. Это самый быстрый путь для создания типового сервера;
- **Наследование**: создайте свой процессор, унаследовавшись от `RadiusProcessorBase`, 
это даст вам готовую обвязку и позволит сфокусироваться только на логике обработки пакетов;
- **Полный контроль**: реализуйте интерфейс `IRadiusProcessor` самостоятельно,
если вам требуется нестандартное поведение на низком уровне.

### 3. Запуск и настройка
После настройки DI и регистрации процессора сервер автоматически начнет слушать входящие UDP-запросы.
Порты и сетевые параметры указываются в стандартном файле конфигурации `appsettings.json`:
```json
{
  "RadiusListener": {
    "BindAddress": "0.0.0.0",
    "Ports": [1812, 1813],
    "InboundQueueSize": 1000,
    "ConcurrentWorkers": 2
  },
  "RadiusSender": {
    "BindAddress": "0.0.0.0",
    "Port": 3799,
    "DefaultTimeout": "00:00:03"
  },
  "RadiusIsp": {
    "NasTimeout": "00:00:05",
    "FramedMtu": 1492,
    "SessionTimeout": 86400,
    "IdleTimeout": 3600,
    "AcctInterimInterval": 300,
    "ReplyMessage": "Welcome to the MoRFaiR Network!"
  },
  "DemoSecurity": {
    "SharedSecret": "testing123"
  }
}
```

В директории [src/MF.Radius.SampleServer](https://github.com/morfair/MF.Radius/tree/main/src/MF.Radius.SampleServer) вы найдете пример полноценного сервера
с интеграцией REST API, логов и метрик.
